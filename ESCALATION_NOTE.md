# Advisor Escalation Note

Use this note when escalation to a stronger advisor model is needed.

## Ask For
- Top root-cause hypotheses
- Best next diagnostic step
- Preferred design direction
- Main risks and constraints
- Compact execution plan

## Keep It Short
- Summarize the issue first.
- Include only the smallest relevant logs or code snippets.
- Avoid long raw dumps unless they are necessary to answer the question.

## Escalation Triggers
- Same issue failed after two attempts
- Root cause is unclear
- Design tradeoffs are significant
- Side effects may be broad
- Migration or integration risk is high
